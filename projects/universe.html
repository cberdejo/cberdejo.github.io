Universe-Simulation-Java

A 2D gravitational particle simulation using Newtonian mechanics

View on GitHub

Project Overview

Purpose

Simulate and visualize the motion of particles under gravity in a 2D universe.

Key Features

  • Real-time 2D simulation with GUI
  • Object-oriented design separating geometry, physics, and GUI
  • Reads initial conditions from text files
  • Scalable rendering for any universe size

Main Classes

1. Geometry/Math Classes

lines.Point

Represents a 2D point (x, y) with basic geometric operations.

lines.Vector

Represents a 2D vector, with operations like sum, scalar multiplication, etc.

lines.Line

Represents a line in 2D space for geometric calculations.

lines.Implicit

Used for geometric calculations (not central to the simulation logic).

2. universe.Particle

Physics Core

Represents a single particle in the universe with position, velocity, mass, and radius.

Fields:

  • Point position: Current position
  • Vector velocity: Current velocity
  • double mass: Mass of the particle
  • double radius: Radius (for visualization/collision)

Key Methods:

  • forceFrom(Particle other): Computes the gravitational force exerted by another particle
  • move(Vector force, double dt): Updates position and velocity based on the net force and time step
  • Getters for position, velocity, mass, and radius
  • toString(): String representation for debugging

3. universe.Universe

Simulation Controller

Manages all particles and coordinates the simulation steps.

Fields:

  • Particle[] particles: Array of all particles
  • double radius: Universe radius (for scaling/visualization)

Key Methods:

  • advance(double dt): Advances the simulation by time dt:
    • Calculates net gravitational forces on each particle
    • Updates all positions and velocities
  • particles(): Returns the array of particles
  • radius(): Returns the universe radius

4. gui.UniversePanel

Visualization

The graphical panel for rendering the universe and particles.

Fields:

  • Particle[] parts: Particles to display
  • double scale: Scaling factor for fitting the universe in the window

Key Methods:

  • paint(Particle[] parts, double radius): Sets the particles and scaling, then triggers repaint
  • paintComponent(Graphics g): Draws all particles as circles, scaling and centering them

5. universe.UniverseData

Data Loader

Handles reading the initial universe configuration from input files.

Key Features:

  • Parses text files to create initial particle configurations
  • Supports custom file formats for universe setup
  • Validates input data before simulation begins

How It Works

1

Initialization

Reads a file (see README.md for format) to create particles and universe configuration using DatosUniverso.

2

Simulation Loop

For each time step in Universo:

  • Compute all pairwise gravitational forces
  • Update each particle's velocity and position
  • Repaint the GUI to show new positions

3

Visualization

PanelUniverso draws all particles, scaling them to fit the window with smooth animation.
